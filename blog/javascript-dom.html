What's an analogy to describe the differences between HTML and CSS?

HTML is like a screenplay and CSS is like the set-design and costuming. Then Javascript is the choreography and special effects (lighting, fog, etc).

HTML has certain conventions (like acts or scenes in a play – beginning, middle, end, credits, etc) that describe the content of a page (the words, pictures or items on that page).

CSS is a lot more limitless, as it describes the appearance and formats layouts that are used on a web-page. This would be the equivalent to the backdrop and props and costuming on a play or TV show – it brings the colour and the character and makes it look interesting/good/nice.

Another analogy for the nerdier non-technical person would be HTML is like the genotype (genetic code that describes what a thing is, like human person with a head and body and a tail) and CSS is like the phenotype (genetic code that is expressed as physical features, e.g. blond hair, green eyes, fluffy long-haired curly tail). Then, based on those two things, the human person can do things (like breathing, running, talking, wagging, parkour) which is where Javascript would fit in.

Explain control flow and loops using an example process from everyday life:

Control Flow is the order that a computer reads your code, and is usually from the first line to the last line. An everyday day process that emulates this would be following Google Maps from your house to the grocery store. You set the destination, which gives you a logical route to follow (the control flow is from your house to the store, since we haven’t invented travel portals yet). You start the car (like clicking to load the page, or initiating an event) at your home, then drive toward town.

The sequence (control flow) is important! If you tried to do this out of order, you would never know where the grocery store was, or you may find that your car had no wheels, or that it was actually a bicycle, or that the road to town hadn’t been built yet.

There is a roundabout on the map, so you have to make sure to take the correct exit for the grocery store (the third one, a right turn). As you drive around the roundabout (loop), you (and Google Maps) are looking out for the correct exit (First exit? Nope. Second exit? Nope. Third exit? Yes! Continued roundabout loop is false!). Woohoo, you exited the roundabout (loop) at the right place and continue happily to the grocery store. What a thrill that was! If Google Maps hadn’t told us which exit to take (i.e. there was no condition set) we would have kept on going round and round that roundabout until we starved to death or crashed...

The for is a conditional loop, which means that it runs based on if a certain condition is true. As long as it stays true, the code is going to keep executing.

This was an example of a ‘for’ loop – which goes for a specified duration or until the condition is false. We intend to drive only once around the roundabout looking for the third exit, because we know when it’s coming up (thanks to Google Maps). We are looking out for the third exit, so would keep driving until we either took the third exit or missed it (the loop stopped after there were no more exits to look for).

An example of the ‘while’ loop would be to drive around and around the roundabout until you’re sure that Google Maps was right and trustworthy and that the exit you need is actually the third one (because the sign on the road says so, but it took you five circuits of the roundabout to make out the tiny road sign in the distance). We are looking for the correct exit while going around the roundabout (but we don’t know when we’ll see it).

An example of the ‘do-while’ loop would be that you already know you have to take the third exit (because Google Maps said so). You only have to do the driving on the roundabout while you get to the third exit. If you could just turn right immediately you would, but since the roundabout exists, you’re forced to go around it at least once to get there.

So, in summary, loops are repetitive processes (statements) that will keep running until there is either nothing left to loop over, or until the condition becomes false. For as long as the given condition is true, it will keep executing.

Describe what the DOM is and an example of how you might interact with it.

The DOM (Document Object Model) is a tool that can be used to look at the content and features of web pages. It allows access to all the elements in html, css and javascript, and lets you adjust live within the browser.

Explain the difference between accessing data from arrays and objects.

Objects represent a special data type that is mutable and can be used to store a collection of data (rather than just a single value). Arrays are a special type of variable that is also mutable and can also be used to store a list of values. Objects represent “things” with characteristics (aka properties), while arrays create and store lists of data in a single variable.

Explain what functions are and why they are useful.

Functions allow you to store a piece of code that does an activity inside a defined block and then call that code whenever you need it using a single short command, rather than having to type out the same code multiple times.
